# Quickstart

## Core controls

- `j` / `↓` move down
- `k` / `↑` move up
- `[` / `]` switch inspector views
- `p` or `:` open the command palette
- `a` / `s` move through actions in the Actions view
- `Enter` run or confirm
- `Esc` cancel a confirmation or close the palette
- `q` quit

## Main regions

The interface is built around four stable regions:

- interface list
- inspector
- event log
- status bar

That keeps movement simple and makes state easy to track while you work.

A good first pass in v1 is:
- select an interface
- check Overview for route and service context
- switch to Signals to inspect reachability posture and default-route state
- use Actions only after the interface story is clear
