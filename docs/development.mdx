# Development

## Repository structure

The v1 repository is intentionally small and modular.

- `src/main.rs`: startup entry point
- `src/lib.rs`: reusable crate surface
- `src/app.rs`: state model and interaction behavior
- `src/network.rs`: interface data model and macOS discovery logic
- `src/tui.rs`: rendering and event loop
- `tests/`: behavioral and parsing coverage
- `docs/`: Mintlify documentation

## Constraints for a network interface CLI

A macOS network interface tool should be conservative.

That means:

1. inspect first
2. avoid hidden mutation
3. make state visible before action
4. require explicit confirmation before disruptive operations
5. degrade gracefully when system-specific data is unavailable

## Current v1 posture

The current app prefers real macOS discovery when available and falls back to a deterministic sample model when it is not.

That gives us a stable operator surface while the mutation layer is still being developed.
