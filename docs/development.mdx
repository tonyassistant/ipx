# Development

## Current phase

ipx is being rebuilt from a clean slate.

That means we are intentionally focusing on:
- good documentation structure
- product clarity
- a clean foundation before implementation expands again

## Principles

1. keep the interface keyboard-first
2. prefer operational clarity over visual noise
3. document intent early
4. make risky actions explicit
5. design for engineers, not demos

## Near-term priorities

- establish polished docs
- define v1 scope clearly
- rebuild implementation against that scope
