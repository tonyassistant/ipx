# Inspector

The inspector is the main detail surface for the selected interface.

It is split into three views.

## Overview

Shows the selected interface in a practical format:

- interface name
- device name
- type
- status
- IPv4 address
- MAC address
- related services when available on the current platform
- notes gathered during discovery

## Signals

The Signals view shows live posture and route context for the selected interface.

In v1 it currently highlights:

- link state
- reachability posture
- IPv4 address
- gateway presence
- default-route context
- service-binding count
- platform-specific discovery notes

Reachability is intentionally conservative:
- `reachable` means the interface looks globally routed
- `private route` means a default route exists, but the upstream is still private
- `local only` means there is an address without a credible upstream path
- `down` and `unknown` remain explicit

## Actions

The Actions view shows the available operations for the selected interface, together with their safety posture.
