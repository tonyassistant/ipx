# Inspector

The inspector is the main detail surface for the selected interface.

It is split into three views.

## Overview

Shows the selected interface in a practical format:

- interface name
- device name
- type
- status
- IPv4 address
- MAC address
- related services when available on the current platform
- notes gathered during discovery

## Signals

The Signals view reserves space for link and signal-oriented surfaces.

In v1, it acts as a structured place for that future detail without overloading the main overview.

## Actions

The Actions view shows the available operations for the selected interface, together with their safety posture.
