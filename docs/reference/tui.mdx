# TUI reference

## Main regions

The current interface is split into four regions:

1. header
2. interface list
3. inspector with tabs
4. event log and status footer

## Inspector tabs

### Overview
Shows identity, status, addressing, and notes.

### Signals
Reserved for signal and link-quality surfaces.

### Actions
Reserved for explicit action entry points and future confirmations.

## Command palette

The command palette is the central interaction surface for non-navigation commands.

Current v1 commands:

- `refresh`
- `reload`
- `help`
- `copy`
- `inspect`
- `quit`

## Navigation model

The TUI is designed to feel deliberate and operator-grade:

- list movement stays lightweight
- inspector context is always visible
- event log captures intent and action history
- footer keeps commands discoverable
